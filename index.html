Data Pipeline

The sync cycle executes every 5 minutes. A full fleet cycle completes in under 30 seconds. Data flows through six stages: discovery, acquisition, normalization, persistence, aggregation, and broadcast.

Solarman API
→
Sync Engine
→
MongoDB
→
Dashboard

5-minute cycle → normalize → upsert → aggregate → broadcast via SSE

Pipeline Stages

1
Plant Discovery

GET /station/search

Enumerates all registered stations. Maintains registry for incremental sync.

2
Real-Time Metrics

POST /station/realtime

Per-plant: generation power, daily yield, battery SOC, grid import/export, load.

3
Device Inventory

POST /device/page

Per-station: inverters, dataloggers, meters with serial numbers and firmware versions.

4
Device Telemetry

POST /device/currentData

Per-device: voltage, current, power, temperature, fault codes, operational status.

5
Schema Normalization

Unifies disparate field names, units, and response structures into a single canonical schema before storage.

6
Upsert & Broadcast

Idempotent writes to MongoDB. On-write rollups for hourly, daily, monthly, yearly views. SSE push to all connected clients.

Risk Mitigation & Safeguards

The system is designed to degrade gracefully under failure conditions. No single point of failure blocks the entire fleet. Each safeguard is implemented and tested.

Risk FactorMitigationSeverity
Solarman token expiration Multi-source auto-extraction with heuristic detection and graceful fallback to secondary tokens Critical
API rate limiting (15 req/min) Intelligent request queuing with staggered dispatch and token pool rotation High
Inconsistent API responses Schema normalizer maps all endpoint variants to a single canonical model High
Network interruption Exponential backoff retry with partial success persistence — never lose partial data Moderate
Device timestamp drift Server-side time anchoring — all records normalized to UTC at ingestion Moderate
Duplicate records Idempotent upserts — safe to re-run any sync cycle without data corruption Moderate
Live vs. history query conflict In-memory hot cache for live data; indexed cold storage for historical queries Moderate
Single plant failure cascades Per-plant error isolation — one failed feed does not block the sync engine Moderate
